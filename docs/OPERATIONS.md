# Обновление и публикация

## Где менять код

- вопросы и логика формы: `app.js`;
- внешний вид и мобильная версия: `styles.css`;
- каркас страницы: `index.html`;
- сохранение, таблица, статусы и уведомления: `google-apps-script/Code.gs`.

`google-apps-script/Index.html` генерируется командой сборки. Не редактируйте его вручную.

## Как тестировать

```bash
npm test
```

После локальной проверки обязательно проверить опубликованную форму:

1. открыть главную и конкретный бриф на компьютере;
2. проверить форму в мобильном размере;
3. убедиться, что без обязательных полей и согласия отправка невозможна;
4. отправить тестовую заявку;
5. проверить новую строку со статусом `Новая`;
6. изменить статус, ответственного, комментарий, amo-ссылку и следующий шаг;
7. открыть Google Doc с полными ответами;
8. проверить уведомление Slack или Telegram;
9. проверить ссылку на политику обработки данных;
10. проверить `brief.serenity.agency`.

## Как публиковать Apps Script

1. Запустить `npm test`.
2. Скопировать `google-apps-script/Code.gs` в файл `Код.gs`.
3. Скопировать `google-apps-script/Index.html` в файл `Index.html`.
4. Сохранить проект.
5. При изменениях реестра запустить `setupRegistry`.
6. Открыть `Deploy → Manage deployments`.
7. Отредактировать рабочее Web app-развертывание и выбрать `New version`.
8. Оставить `Execute as: Me`, `Who has access: Anyone`.
9. Проверить публичную ссылку и отправить тестовую заявку.

## Работа через GitHub

- каждая задача выполняется в отдельной ветке;
- перед слиянием запускается `npm test` и полный тест опубликованной формы;
- в коммите не должно быть ID таблиц, токенов, webhook и клиентских ответов;
- рабочая версия помечается релизом или тегом после успешной публикации;
- откат: развернуть предыдущую версию Apps Script и вернуть соответствующий коммит.

## Брендовый frontend без Google-баннера

Клиентская форма публикуется в Cloudflare Pages, а Apps Script используется только как закрытый обработчик заявок.

1. Создайте Pages-проект из репозитория.
2. Build command: `npm run build:pages`.
3. Build output directory: `dist`.
4. Добавьте переменные `APPS_SCRIPT_URL` и `FORM_API_SECRET`.
5. Добавьте такой же `FORM_API_SECRET` в Apps Script → Script Properties.
6. Опубликуйте новую версию Apps Script с функцией `doPost`.
7. Подключите к Pages домен `brief.serenity.agency`.
8. Откройте форму на брендовом домене и отправьте тестовую заявку.

Редирект на Apps Script больше не используется: иначе техническая плашка Google останется видимой.

## Текущая публикация

- Pages-проект: `serenity-brief`;
- резервный публичный адрес: `https://serenity-brief.pages.dev`;
- брендовый адрес: `https://brief.serenity.agency`;
- Apps Script используется только для серверного сохранения заявок.

Для внешнего DNS-провайдера создайте CNAME:

| Поле | Значение |
| --- | --- |
| Имя | `brief` |
| Назначение | `serenity-brief.pages.dev.` |

В REG.RU у назначения нужна точка в конце. Без неё сервис может превратить адрес в
`serenity-brief.pages.dev.serenity.agency`, и поддомен не будет открываться.

После изменения проверьте:

```bash
dig +short brief.serenity.agency CNAME
curl -I https://brief.serenity.agency
```

Ожидаемый CNAME: `serenity-brief.pages.dev.`. Затем в Cloudflare Pages откройте
`Custom domains` и дождитесь статуса `Active`.
