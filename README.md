# Serenity Brief

Публичная форма брифов Serenity. Клиент выбирает услугу, заполняет подходящий бриф без регистрации, а ответы сохраняются в Google Drive и в рабочий реестр Google Sheets.

## Что делает проект

- показывает 12 отдельных брифов под разные услуги;
- сохраняет черновик в браузере клиента;
- требует согласие на обработку персональных данных;
- создаёт Google Doc с полными ответами;
- добавляет краткую карточку заявки в Google Sheets;
- ставит новой заявке статус `Новая`;
- может отправлять уведомление в Slack или Telegram;
- защищает форму от дублей, слишком быстрых и частых отправок.
- открывается на брендовом Cloudflare Pages frontend без технической плашки Google.

## Куда сохраняются ответы

Apps Script создаёт отдельный Google Doc для каждой отправки и переносит его в папку из свойства `RESPONSES_FOLDER_ID`. Краткие данные добавляются в таблицу из свойства `REGISTRY_SPREADSHEET_ID`.

В репозитории нет ID папки, ID таблицы, webhook, токенов и других закрытых настроек.

## Поля реестра Google Sheets

`Дата заявки`, `Компания`, `Имя`, `Контакт`, `Мессенджер`, `Задача`, `Бюджет`, `Статус`, `Ответственный`, `Комментарий менеджера`, `Ссылка на сделку в amo`, `Дата последнего касания`, `Следующий шаг`, `Бриф`, `Google Doc`, `Согласие получено`.

Технические поля `ID отправки` и `Ответы JSON` скрыты. В основной таблице включён фильтр. Скрипт также создаёт представления: `Новые`, `Мои заявки`, `Без ответственного`, `Без движения`, `КП отправлено`. В `Мои заявки` менеджер выбирает себя в поле `Ответственный`; `Без движения` показывает активные назначенные заявки без касания или следующего шага.

Статусы: `Новая`, `Взята в работу`, `Уточняем`, `Передано специалисту`, `КП готовится`, `КП отправлено`, `Встреча назначена`, `Не целевой`, `Закрыто`.

## Уведомления

Уведомления подключаются через закрытые свойства Apps Script:

- Slack: `NOTIFICATION_PROVIDER=slack` и `SLACK_WEBHOOK_URL`;
- Telegram: `NOTIFICATION_PROVIDER=telegram`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`.

После настройки запустите функцию `testNotification` в Apps Script. Ошибка уведомления не мешает сохранить заявку.

## Доступы

Нужны:

- доступ редактора к Apps Script;
- доступ редактора к папке ответов и таблице;
- доступ к DNS домена для настройки `brief.serenity.agency`;
- доступ к Cloudflare Pages для публикации брендового frontend;
- webhook Slack или Telegram Bot Token и Chat ID, если нужны уведомления;
- утверждённая публичная ссылка на политику обработки данных.

Все закрытые значения хранятся только в **Project Settings → Script Properties**. Полный список есть в [docs/SCRIPT_PROPERTIES.md](docs/SCRIPT_PROPERTIES.md).

## Быстрое обновление

1. Измените `app.js`, `styles.css`, `index.html` или `google-apps-script/Code.gs`.
2. Запустите `npm test`.
3. Перенесите собранный `google-apps-script/Index.html` и `Code.gs` в Apps Script.
4. Запустите `setupRegistry`, если менялась таблица или статусы.
5. Создайте новую версию развертывания Web app.
6. Опубликуйте Cloudflare Pages frontend.
7. Отправьте тестовую заявку и проверьте реестр, документ и уведомление.
8. Зафиксируйте изменения отдельным коммитом и опишите, что проверено.

Подробный процесс: [docs/OPERATIONS.md](docs/OPERATIONS.md).
